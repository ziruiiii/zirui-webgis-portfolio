What the map can be used for

This interactive Web App is designed for outdoor travellers who are planning for winter visits to parks across the United States, by visualizing and forecasting the snowfall intensity in the next 72 hours. The primary purpose is to help outdoor travellers to identify potential snowfall hazards in real-time.

How to use it

  • Use the Geocoding Service to search a specific destination (e.g., “Yosemite” or “Glacier National Park”).
  • Move the Time Slider at the bottom to animate snowfall intensity changes in 6-hour intervals across a 72-hour forecast.
  • Use the Bookmarks can be used to jump between the "Eastern US", "Western US", and nationwide windows.

Methodology

Built as an ArcGIS Instant App with the "Slider" template, which helps to animate and forecast over a 72-hour with a 6-hour interval projection. To reduce visual clutter and contrast against a dark canvas basemap, the USA Parks dataset was simplified into two categories: "Major Parks + Forests" and "Local & Regional Parks".

What the map can be used for

The ArcGIS dashboard “Global Maritime Security Landscape (2015-2024) visualizes global shipping routes with reported maritime security incidents. It provides an overview of where incidents concentrate, how the incident pattern changes over time, and the common incidents types over the world.

How to use it

  • Pan and Zoom, or use the Bookmarks to explore hotspots near major shipping corridors (e.g., Strait of Malacca, Red Sea).
  • Use the Pie Chart to explore the distribution of incident types.
  • Interactive Filter: Click a bar on the serial chart (a specific year) to filter the map and pie chart to that year. And click again to clear.

Methodology (How it was created)

This application was developed in two main phases using ArcGIS Online Map Viewer and ArcGIS Dashboards:

  • Data Processing (in Map Viewer): The web map was first built by overlaying the global shipping routes and maritime incident points. For the layer of maritime incident points, the original incident type field (“hostility_”) contained highly specific categories. An Arcade expression was used to create a simplified “Incident Group” field, combining events into Piracy/Boarding, Armed Attack, Suspicious Activity, Protest, and Other.
  • Dashboard Configuration: The layout was assembled using ArcGIS Dashboards. The serial chart was integrated using the "Add element" function, where the incident date field was parsed to categorize the X-axis by year.
  • Interactive Actions: To create the dynamic filter, dashboard "Actions" were configured. By setting the serial chart as the source, clicking a specific bar (representing a year) triggers a filter action that updates both the map and the pie chart to show only the incidents from that selected timeframe.
  • Map Elements: Scale Bar and Legends are implemented for inviting users to better understand the scale of the map, and the symbology representation across two web map apps.

🎯 Design Rationale: Selection of Web Map Applications

Template choices and interface design

❄️ Instant App (Slider Template)

The snowfall application mainly focuses on forecast snowfall intensity. Thus, the primary analytical goal is temporal visualization, rather than multi-variable comparison.

The Slider template in ArcGIS Instant Apps supports time-enabled layers, allowing 6-hour interval animation across a 72-hour forecast window.

The interface is designed to be lightweight and guided, aligning with the needs of outdoor travellers who require quick situational awareness.

🚢 ArcGIS Dashboard

The maritime security dataset contains multiple attributes including incident year, hostility category, geographic clustering, and proximity to global shipping corridors.

ArcGIS Dashboards supports action-based filtering between charts and map layers. This enables dynamic interaction, where selecting a specific year immediately updates both spatial and categorical visualizations.

The dashboard tend to prioritize analytical depth and cross-element interaction, which would not be achievable within a single-template Instant App structure.

Data Sources & Limitations

An overview of the spatial data and their technical constraints.

❄️ Snowfall Explorer

Data Acquisition and Processing

Data Source:

USA Parks layer (used for park boundaries and classification):
https://msc2023.maps.arcgis.com/home/item.html?id=e49e181ac82c46edac3ae601ebb3ef2d

Snowfall Forecast layer (72-hour forecast, 6-hour interval):
https://msc2023.maps.arcgis.com/home/item.html?id=be1bb766bf1c44a9be97bbb7c04355ff

These layers were first assembled in ArcGIS Online Map Viewer to create the base web map. The web map was then configured using the Instant App (Slider template) to enable temporal animation of snowfall intensity across the forecast window.

Technical Limitations

A technical limitation exists regarding the temporal synchronization of the snowfall data. The live feed from the National Weather Service currently remains static at the forecast period of Mar 1 to Mar 3, 2026, and the automatic refresh function could not be enabled. This discrepancy between the live service and saved configuration creates a synchronization gap after 72 hours.

🚢 Maritime Dashboard

Data Acquisition and Processing

Data Source:

Global Shipping Routes Dataset:
https://msc2023.maps.arcgis.com/home/item.html?id=f5557ba2eb3f493dafe6b8b5bff373e3
This dataset represents global shipping intensity derived from vessel movement records. It provides a spatial estimate of major maritime corridors and global trade routes.

Maritime Security Incident Dataset (ASAM):
https://msc2023.maps.arcgis.com/home/item.html?id=fdcbea85edc5446da0bbc6c913fcd46e
This dataset contains reported hostile acts against ships worldwide, including incident date, geographic coordinates, and descriptive hostility categories.

In the Dashboard, the global shipping routes layer functions as a spatial context layer, highlighting major shipping corridors, and enabling comparison between traffic intensity and incident concentration. The ASAM dataset helps to derive the serial chart, pie chart, and year-based interactive filtering through configured dashboard actions. Overall, those information set the foundation for the further maritime safety analysis.

Technical Limitations

The global shipping routes layer is a tile layer, thus its symbology and color ramp could not be amended within ArcGIS Online. If a feature layer-based version was available, a grey to light blue gradient would be used to represent traffic density, ensuring shipping routes remain visible while keeping incident hotspots as the primary visual focus.
